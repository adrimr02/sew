Caracteristicas de Node.js

Sistema de paquetes de Node

Las principales funcionalidades de Node vienen recogidas en varios modulos que pueden ser importados cuando se necesiten (mas adelante se veran los mas importantes) y que ya vienen instalados por defecto. Sin embargo, muchas veces es necesaria cierta funcionalidad que seria muy dificil implementar desde cero, por lo que se pueden importar modulos nuevos usando el gestor de paquetes npm.

Npm se instala automaticamente al instalar Node y es una herramienta que se utiliza desde la consola. Aunque su nombre indica que es un gestor de paquetes, realmente hace mucho mas, ya que permite, entre muchas otras cosas, crear comandos, guardar la configuracion de los proyectos y añadir o importar ejecutables creados con Node para realizar una tarea dentro del propio proyecto, como la generación automatica de código.

Modulos incluidos con Node

La lista de modulos que trae Node por defecto es bastante larga, pero a continuación están los mas importantes:

fs
Permite utilizar el sistema de archivos de la maquina. Leer y crear archivos, carpetas, etc
http
Permite utilizar Node.js como un servidor HTTP. Tambien existe el modulo HTTPS que es similar pero con la seguridad extra
crypto
Añade la funcionalidad para encriptar informacion o para crear claves hash, entre otras muchas cosas relacionadas
path
Se utiliza principalmente para crear direcciones a archivos relativas al proyecto de node, y que sean independiente a donde se encuentre este proyecto. Aunque tiene otras funciones relacionadas con el manejo de direcciones de archivos dentro del sistema.
os
Se usa sobretodo para conseguir informacion sobre el sistema en el que se ejecuta el programa. Infomacion sobre la arquitectura, el numero de CPUs, el nombre y version del sistema operativo, entre otras.
buffer
Se utiliza para almacenar y procesar flujos de datos binarios, o para convertir otros datos en datos binarios o viceversa. Es de gran utilidad para mover cantidades grandes de datos.
cluster
Es el modulo que se utiliza para dividir un proceso de Node en varios subprocesos que se ejecuten simultaneamente utilizando distintas CPUs.
